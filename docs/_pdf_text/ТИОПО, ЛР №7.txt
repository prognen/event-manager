Примечания
  1.​ Можно использовать любой из проектов, который был использован в
      лабораторных 1-6
  2.​ Mock-сервера нужны в разных случаях:
         a.​ основной проект тестируется и разрабатывается в закрытой сети
         b.​ внешний сервис требует закупки прав, невозможной на этапе разработки
             и тестирования
         c.​ внешний сервис требует аппаратных ресурсов, которые недоступны
             разработчику
         d.​ внешний сервис ещё не выбран, но есть несколько кандидатов с
             похожим контрактом взаимодействия
         e.​ внешний сервис требует использования реальных платёжных средств
             при работе с ним
         f.​ всё сразу

Лабораторная работа № 7

Задание
  1.​ Добавить в выбранный проект сценарий интеграции с любым внешним
      сервисом, который для студента является “чёрным ящиком”, но для которого
      описан контракт взаимодействия (контракт может быть описан в любом
      формате – .wsdl, .proto, swagger, официальная документация с примерами)
  2.​ Реализовать сценарий интеграции в коде проекта
  3.​ Реализовать Е2Е-сценарий тестирования этой интеграции
  4.​ Реализовать mock-сервер внешнего сервиса
  5.​ Настроить пайплайн выполнения тестового Е2Е-сценария с mock-сервером и
      реальным сервисом
  6.​ Настроить запуск окружения для демонстрации этого сценария с
      mock-сервером и реальным сервисом

Требования
  1.​ Внешний сервис может быть чем угодно – любой из сервисов yandex cloud, vk
      cloud, amazon web services, любая платёжная система, любая внешняя llm и т.д.
      – согласовать с преподавателем
  2.​ Как любая внесистемная зависимость mock-сервер указывается в
      конфигурации проекта, через эту конфигурацию должно быть реализовано
      переключение режима работы – с mock-сервером / с реальным сервисом
